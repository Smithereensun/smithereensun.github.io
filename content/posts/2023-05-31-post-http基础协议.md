<h1 style="text-align: center">浏览器请求方法</h1>
<ul>
<li>
<p>http1.0定义了三种：</p>
<ul>
<li><strong><span style="color: rgba(255, 0, 0, 1)">GET</span></strong>: 向服务器获取资源，比如常见的查询请求</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">POST</span></strong>: 向服务器提交数据而发送的请求</li>
<li>Head: 和get类似，返回的响应中没有具体的内容，用于获取报头</li>
</ul>
</li>
<li>
<p>http1.1定义了六种</p>
<ul>
<li><strong><span style="color: rgba(255, 0, 0, 1)">PUT</span></strong>：一般是用于更新请求，比如更新个人信息、商品信息全量更新</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">PATCH</span></strong>：PUT 方法的补充,更新指定资源的部分数据</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">DELETE</span></strong>：用于删除指定的资源</li>
<li>OPTIONS: 获取服务器支持的HTTP请求方法,服务器性能、跨域检查等</li>
<li>CONNECT: 方法的作用就是把服务器作为跳板，让服务器代替用户去访问其它网页，之后把数据原原本本的返回给用户，网页开发基本不用这个方法，如果是http代理就会使用这个，让服务器代理用户去访问其他网页，类似中介</li>
<li>TRACE：回显服务器收到的请求，主要用于测试或诊断</li>
</ul>
</li>
</ul>
<h1 style="text-align: center">Http响应码</h1>
<ul>
<li>浏览器向服务器请求时，服务端响应的消息头里面有状态码，表示请求结果的状态</li>
</ul>
<ul>
<li>
<p>分类</p>
<ul>
<li>
<p>1XX: 收到请求，需要请求者继续执行操作，比较少用</p>
</li>
<li>
<p>2XX: 请求成功，常用的 200</p>
</li>
<li>
<p>3XX: 重定向，浏览器在拿到服务器返回的这个状态码后会自动跳转到一个新的URL地址，这个地址可以从响应的Location首部中获取；</p>
<ul>
<li>
<p>好处：网站改版、域名迁移等，多个域名指向同个主站导流</p>
</li>
<li>
<p>必须记住</p>
<ul>
<li><strong><span style="color: rgba(255, 0, 0, 1)">301</span></strong>：永久性跳转，比如域名过期，换个域名</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">302</span></strong>：临时性跳转</li>
</ul>
</li>
</ul>
</li>
<li>
<p>4XX: 客户端出错，请求包含语法错误或者无法完成请求</p>
<ul>
<li>
<p>必须记住</p>
<ul>
<li><strong><span style="color: rgba(255, 0, 0, 1)">400</span></strong>: 请求出错，比如语法协议</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">403</span></strong>: 没权限访问</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">404</span></strong>: 找不到这个路径对应的接口或者文件</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">405</span></strong>: 不允许此方法进行提交，Method not allowed，比如接口一定要POST方式，而你是用了GET</li>
</ul>
</li>
</ul>
</li>
<li>
<p>5XX: 服务端出错，服务器在处理请求的过程中发生了错误</p>
<ul>
<li>
<p>必须记住</p>
<ul>
<li><strong><span style="color: rgba(255, 0, 0, 1)">500</span></strong>: 服务器内部报错了，完成不了这次请求</li>
<li><strong><span style="color: rgba(255, 0, 0, 1)">503</span></strong>: 服务器宕机</li>
</ul>
</li>
</ul>
</li>
</ul>
</li>
</ul>
<h2>口诀</h2>

<h1 style="text-align: center"><strong>http常见请求头</strong></h1>
<ul>
<li>
<p>http请求分为三部分：请求行，请求头， 请求体</p>
</li>
<li>
<p>请求头</p>
<ul>
<li>报文头包含若干个属性 格式为“属性名:属性值”，</li>
<li>服务端据此获取客户端的基本信息</li>
</ul>
</li>
<li>
<p>常见的请求头</p>
<ul>
<li>
<p>Accept： 览器支持的 MIME 媒体类型, 比如 text/html,application/json,image/webp,<em>/</em>&nbsp;等</p>
</li>
<li>
<p>Accept-Encoding: 浏览器发给服务器,声明浏览器支持的编码类型，gzip, deflate</p>
</li>
<li>
<p>Accept-Language: 客户端接受的语言格式,比如 zh-CN</p>
</li>
<li>
<p>Connection: keep-alive , 开启HTTP持久连接</p>
</li>
<li>
<p>Host：服务器的域名</p>
</li>
<li>
<p>Origin：告诉服务器请求从哪里发起的，仅包括协议和域名 CORS跨域请求中可以看到response有对应的header，Access-Control-Allow-Origin</p>
</li>
<li>
<p>Referer：告诉服务器请求的原始资源的URI，其用于所有类型的请求，并且包括：协议+域名+查询参数； 很多抢购服务会用这个做限制，必须通过某个入来进来才有效</p>
</li>
<li>
<p>User-Agent: 服务器通过这个请求头判断用户的软件的应用类型、操作系统、软件开发商以及版本号、浏览器内核信息等； 风控系统、反作弊系统、反爬虫系统等基本会采集这类信息做参考</p>
</li>
<li>
<p>Cookie: 表示服务端给客户端传的http请求状态,也是多个key=value形式组合，比如登录后的令牌等</p>
</li>
<li>
<p><strong>Content-Type</strong>： HTTP请求提交的内容类型，post提交时才需要设置，比如文件上传，表单提交、json等</p>
<ul>
<li>form表单提交：application/x-www-form-urlencoded</li>
<li>json方式提交：application/json</li>
</ul>
</li>
</ul>
</li>
</ul>
<h1 style="text-align: center"><strong>Http响应头</strong></h1>
<ul>
<li>
<p>响应头</p>
<ul>
<li>报文头包含若干个属性 格式为“属性名:属性值”</li>
</ul>
</li>
<li>
<p>常见的响应头</p>
<ul>
<li>Allow: 服务器支持哪些请求方法</li>
<li>Content-Length: 响应体的字节长度</li>
<li>Content-Type: 响应体的MIME类型</li>
<li>Content-Encoding: 设置数据使用的编码类型</li>
<li>Date: 设置消息发送的日期和时间</li>
<li>Expires: 设置响应体的过期时间,一个GMT时间，表示该缓存的有效时间</li>
<li>cache-control: Expires的作用一致，都是指明当前资源的有效期, 控制浏览器是否直接从浏览器缓存取数据还是重新发请求到服务器取数据,优先级高于Expires,控制粒度更细，如max-age=240，即4分钟</li>
<li>Location：表示客户应当到哪里去获取资源，一般同时设置状态代码为3xx</li>
<li>Server: 服务器名称</li>
<li>Transfer-Encoding：chunked 表示输出的内容长度不能确定，静态网页一般没，基本出现在动态网页里面</li>
<li>Access-Control-Allow-Origin: 定哪些站点可以参与跨站资源共享</li>
</ul>
</li>
</ul>
<h1 style="text-align: center"><strong>Http里面的content-type</strong></h1>
<ul>
<li>
<p>Content-type: 用来指定不同格式的请求响应信息，俗称 MIME媒体类型</p>
</li>
<li>
<p>常见的取值</p>
<ul>
<li>text/html ：HTML格式</li>
<li>text/plain ：纯文本格式</li>
<li>text/xml ： XML格式</li>
<li>image/gif ：gif图片格式</li>
<li>image/jpeg ：jpg图片格式</li>
<li>image/png：png图片格式</li>
<li>application/json：JSON数据格式</li>
<li>application/pdf ：pdf格式</li>
<li>application/octet-stream ：二进制流数据，一般是文件下载</li>
<li>application/x-www-form-urlencoded：form表单默认的提交数据的格式，会编码成key=value格式</li>
<li>multipart/form-data： 表单中需要上传文件的文件格式类型</li>
</ul>
</li>
</ul>
