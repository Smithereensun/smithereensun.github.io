# 为什么要用 nvm？

不同的项目可能会需要不同版本的 Node，特别是接手一些老项目的时候，就需要频繁切换 Node 版本了。

通常来说，一台计算机只能安装一个版本的 Node。如果要运行其他版本，就需要先卸载当前版本，然后安装需要的版本，切来切去的着实比较麻烦。

更糟的情况是：

-

你本地跑着新项目用 Node 18，突然要修一个线上紧急 bug，那个老项目只兼容 Node 14

-

卸载 → 重装 → 跑项目 → 修完 → 再卸载 → 再重装回 Node 18

这一来一回，十几分钟就没了。

而 nvm（Node Version Manager） 就是为了解决这个问题而生的。

它可以让你在一台电脑上同时安装多个 Node 版本，并且能做到：

-

一键切换版本

-

按项目自动指定版本

-

不用反复卸载重装

简单说：nvm = Node 版本的“遥控器”，指哪打哪。

官网地址：[点我直达](https://github.com/coreybutler/nvm-windows/releases)

# 安装node

```text
nvm install 16.20.2 && nvm install 22.22.2 && nvm install latest
```

![image](/imported/posts/2026-04-28-19945673-66117021-windows上安装多版本node/images/img_002_e7eab3bac71e.png)

## 查看已经安装的node版本

```text
nvm list
```

## 切换指定版本

```text
nvm use 22.22.2
```
