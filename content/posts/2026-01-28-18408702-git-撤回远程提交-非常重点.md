<h1 style="text-align: center">IDEA代码撤回办法如下</h1>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911181809455-976504046.png" alt="" loading="lazy" /></p>
<h2 style="text-align: center">例如test123是错误代码，我们需要回撤到test12</h2>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911181844089-254968859.png" alt="" loading="lazy" /></p>
<h1 style="text-align: center">右键点击test12（<span style="color: rgba(255, 0, 0, 1)">选择要回退的版本</span>），选择Reset Current Branch to Here...</h1>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911181909007-950260640.png" alt="" loading="lazy" /></p>
<h2>有以下四种方式回撤代码，这里我们选择<span style="color: rgba(255, 0, 0, 1)">Hard</span></h2>
<p>（1）soft 文件不会更改，差异将暂存提交</p>
<p>（2）Mixed混合文件不会更改，差异不会暂存</p>
<p>（3）<strong><span style="color: rgba(255, 0, 0, 1)">Hard文件将恢复到所选提交的状态警告:任何本地更改都将丢失</span></strong></p>
<p>（4）Keep 文件将恢复到所选提交的状态，但本地更改将保持不变</p>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182001318-993234025.png" alt="" loading="lazy" /></p>
<ul>
<li>这里回退代码之后，git记录变成了白色，本地代码已经变成了test12，然后我要对本地代码进行强制推送，这里我们点击push进行推送，然后点击Force Push即可。</li>
</ul>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182023438-790766003.png" alt="" loading="lazy" /></p>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182043869-946871159.png" alt="" loading="lazy" /></p>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182050916-1816267195.png" alt="" loading="lazy" /></p>
<h2>如果这里<span style="color: rgba(255, 0, 0, 1)">Force Push是灰色</span>，说明无法强制推送，我们要在设置里面清空git里面Protected branches的内容，应用一下，就可以点击Force Push了</h2>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182109759-69565371.png" alt="" loading="lazy" /></p>
<h3>如果<span style="color: rgba(255, 0, 0, 1)">推送被拒绝</span>，则需要在远程仓库，取消该主分支的保护，以giltlab为例，在<a class="hl hl-1" href="https://so.csdn.net/so/search?q=gitlab&amp;spm=1001.2101.3001.7020" target="_blank" data-report-click="{&quot;spm&quot;:&quot;1001.2101.3001.7020&quot;,&quot;dest&quot;:&quot;https://so.csdn.net/so/search?q=gitlab&amp;spm=1001.2101.3001.7020&quot;,&quot;extra&quot;:&quot;{\&quot;searchword\&quot;:\&quot;gitlab\&quot;}&quot;}" data-tit="gitlab" data-pretit="gitlab" rel="noopener nofollow">gitlab</a>的设置里面选择仓库，然后选择Protected branches，然后点击Unprotect，解除保护即可</h3>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182135337-821607610.png" alt="" loading="lazy" /></p>
<p>&nbsp;</p>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182152037-780427750.png" alt="" loading="lazy" /></p>
<p><img src="https://img2024.cnblogs.com/blog/1504448/202409/1504448-20240911182158770-1490836215.png" alt="" loading="lazy" /></p>
<h2>推送完成之后，我们再看一下git日志，发现已经回退到了test12版本了，而且gitlab远程仓库也是test12版本</h2>
