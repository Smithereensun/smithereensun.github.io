<h2>简介(<span style="color: rgba(255, 0, 0, 1)">百度百科</span>)</h2>
<p>　　所谓并发编程是指在一台处理器上“<span style="color: rgba(255, 0, 0, 1)"><strong>同时</strong></span>”<span style="color: rgba(255, 0, 0, 1)"><strong>处理多个任务</strong></span>。并发是在同一实体上的多个事件。多个事件在同一时间间隔发生。</p>
<h2>目标(<span style="color: rgba(255, 0, 0, 1)">百度百科</span>)</h2>
<p>　　并发编程的目标是<span style="color: rgba(255, 0, 0, 1)"><strong>充分</strong></span>的<span style="color: rgba(255, 0, 0, 1)"><strong>利用处理器的每一个核</strong></span>，以达到<span style="color: rgba(255, 0, 0, 1)"><strong>最高</strong></span>的<span style="color: rgba(255, 0, 0, 1)"><strong>处理性能</strong></span>。</p>

<h2>并发编程中的上下文切换</h2>
<p>　　以下内容，<span style="color: rgba(255, 0, 0, 1)"><strong>百度百科原话</strong></span>(<a href="https://baike.baidu.com/item/%E4%B8%8A%E4%B8%8B%E6%96%87%E5%88%87%E6%8D%A2/4842616?fr=aladdin" target="_blank" rel="noopener nofollow">点我直达</a>)。</p>
<p>　　上下文切换指的是<a href="https://baike.baidu.com/item/%E5%86%85%E6%A0%B8/108410" target="_blank" data-lemmaid="108410" rel="noopener nofollow">内核</a>（操作系统的核心）在<a href="https://baike.baidu.com/item/CPU/120556" target="_blank" data-lemmaid="120556" rel="noopener nofollow">CPU</a>上对进程或者线程进行切换。上下文切换过程中的信息被保存在进程控制块（PCB-Process Control Block）中。PCB又被称作切换桢（SwitchFrame）。上下文切换的信息会一直被保存在CPU的内存中，直到被再次使用。<span class="sup--normal" data-sup="1" data-ctrmap=":1,"><br></span></p>
<div class="para">　　上下文切换 (context switch) , 其实际含义是任务切换, 或者CPU寄存器切换。当多任务内核决定运行另外的任务时, 它保存正在运行任务的当前状态, 也就是CPU寄存器中的全部内容。这些内容被保存在任务自己的堆栈中, 入栈工作完成后就把下一个将要运行的任务的当前状况从该任务的栈中重新装入CPU寄存器, 并开始下一个任务的运行, 这一过程就是context switch。
