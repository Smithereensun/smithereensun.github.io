<h1 style="text-align: center">为什么要用 nvm？</h1>
<p class="ds-markdown-paragraph">不同的项目可能会需要不同版本的 Node，特别是接手一些老项目的时候，就需要频繁切换 Node 版本了。</p>
<p class="ds-markdown-paragraph">通常来说，一台计算机只能安装一个版本的 Node。如果要运行其他版本，就需要先卸载当前版本，然后安装需要的版本，切来切去的着实比较麻烦。</p>
<p class="ds-markdown-paragraph">更糟的情况是：</p>
<ul>
<li>
<p class="ds-markdown-paragraph">你本地跑着新项目用 Node 18，突然要修一个线上紧急 bug，那个老项目只兼容 Node 14</p>
</li>
<li>
<p class="ds-markdown-paragraph">卸载 → 重装 → 跑项目 → 修完 → 再卸载 → 再重装回 Node 18</p>
</li>
</ul>
<p class="ds-markdown-paragraph">这一来一回，十几分钟就没了。</p>
<p class="ds-markdown-paragraph">而&nbsp;nvm（Node Version Manager）&nbsp;就是为了解决这个问题而生的。</p>
<p class="ds-markdown-paragraph">它可以让你在一台电脑上同时安装多个 Node 版本，并且能做到：</p>
<ul>
<li>
<p class="ds-markdown-paragraph">一键切换版本</p>
</li>
<li>
<p class="ds-markdown-paragraph">按项目自动指定版本</p>
</li>
<li>
<p class="ds-markdown-paragraph">不用反复卸载重装</p>
</li>
</ul>
<p class="ds-markdown-paragraph">简单说：nvm = Node 版本的“遥控器”，指哪打哪。</p>
