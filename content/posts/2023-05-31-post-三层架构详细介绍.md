<h1><span style="font-family: &quot;Microsoft YaHei&quot;">一、分层架构-3层架构-多层架构</span></h1>
<p><span style="font-family: &quot;Microsoft YaHei&quot;">逻辑关系图</span></p>
<p><span style="font-family: &quot;Microsoft YaHei&quot;"><img src="https://img2018.cnblogs.com/blog/1504448/201905/1504448-20190527234444558-1820968840.png" alt="" /></span></p>
<p>&nbsp;</p>
<p><span style="font-size: 16px; font-family: &quot;Microsoft YaHei&quot;"><strong><span style="color: rgba(255, 0, 255, 1)">架构：</span></strong></span></p>
<p><span style="font-family: &quot;Microsoft YaHei&quot;">　　架构一般是针对整个系统的，并非对某个单独的问题(单独的问题可以用模式等来解决)</span></p>
<p><span style="font-family: &quot;Microsoft YaHei&quot;">　　针对整个系统的“一张蓝图”，<span style="color: rgba(255, 0, 0, 1)"><strong>对系统的抽象。架构与具体的语言平台无关。</strong></span></span></p>
<p><span style="color: rgba(0, 0, 0, 1); font-size: 16px; font-family: &quot;Microsoft YaHei&quot;">　　架构设计、模式应用的经验积累的具体代码实现，方便以后的复用。Asp.net mvc、NHibemate、NSpring、...<br></span></p>
<p><span style="color: rgba(255, 0, 255, 1); font-size: 16px; font-family: &quot;Microsoft YaHei&quot;"><strong>模式:</strong></span></p>
<p>　　软件开发中遇到的一些特定问题，前人总结出来特定的经验、解决方法。(复制某某企业的成功模式)</p>
<p><span style="color: rgba(0, 0, 0, 1)">　　23种设计模式</span></p>
<p><span style="color: rgba(0, 0, 0, 1)">　　MVC、MVP等模式</span></p>
<p><span style="color: rgba(255, 0, 255, 1); font-size: 16px"><strong>理解分层：</strong></span></p>
<p><span style="color: rgba(0, 0, 0, 1)">　　• 逻辑分层<span style="color: rgba(255, 0, 0, 1)"><strong>N-Layer</strong></span></span></p>
<p><span style="color: rgba(0, 0, 0, 1)">　　　　逻辑上将系统中的不同功能模块、不同子系统等进行分层。</span></p>
<p><span style="color: rgba(0, 0, 0, 1)">　　　　好的逻辑分层可以让后续选择物理架构更灵活，选择性更大</span></p>
<p><span style="color: rgba(0, 0, 0, 1)">　　• 物理分层<span style="color: rgba(255, 0, 0, 1)"><strong>N-Tier</strong></span></span></p>
<p><span style="color: rgba(0, 0, 0, 1)">　　　　物理部署时将系统的不同模式部署在不同的服务器上</span></p>
<p><span style="color: rgba(0, 0, 0, 1)">一句话总结架构：项目的组成、分布，什么问题该怎么处理(对于一些关键性问题的预见性与解决方法)。对整个项目的规划、设计，以及在一个系统中各个组件间的组合、交互、集成。架构保证了系统的可用性、稳定性、灵活性、可伸缩性、安全性等等。</span></p>
