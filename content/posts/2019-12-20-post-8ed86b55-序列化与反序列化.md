<p><span style="font-family: 楷体">　　对象序列化：是将对象转换为二进制数据</span></p>
<p><span style="font-family: 楷体">　　反序列化：是将二进制数据还原为对象。</span></p>
<p><span style="font-family: 楷体">　　对象是稍纵即逝的，不仅程序重启、操作系统重启会造成对象的消失，就是退出函数范围范围等都可能造成对象的消失，<span style="color: rgba(255, 0, 0, 1)"><strong>序列化/反序列化</strong></span>就是为了<strong><span style="color: rgba(255, 0, 0, 1)">保持对象</span></strong>的<span style="color: rgba(255, 0, 0, 1)"><strong>持久化</strong></span>。</span></p>
<p><span style="font-family: 楷体">　　<span style="color: rgba(255, 0, 0, 1)"><strong>对象序列化，只能针对对象的字段进行序列化。</strong></span></span></p>
<p><span style="color: rgba(0, 0, 0, 1); font-family: 楷体">BinaryFormatter类有两个方法：</span></p>
<p><span style="color: rgba(0, 0, 0, 1); font-family: 楷体">　　• void Serialize(Stream stream,object graph) 对象graph序列化到stream中</span></p>
<p><span style="color: rgba(0, 0, 0, 1); font-family: 楷体">　　• object Deserialize(Stream stream) 将对象从stream中反序列化，返回值为反序列化得到的对象。</span></p>
<p><span style="color: rgba(255, 0, 255, 1); font-family: 楷体"><strong>注：不是所有对象都能序列化，只有可序列化的对象才能序列化，<span style="color: rgba(255, 0, 0, 1)">在类的声明上加[Serializable]，对象的属性、字段的类型也必须可序列化</span></strong></span></p>
